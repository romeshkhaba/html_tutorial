HTML SEMANTIC LAYOUT

Semantic layout tags describe the purpose of each part of a page. They make code easier to read and help browsers, search engines, and screen readers understand the page structure.

Important semantic tags include <header>, <nav>, <main>, <section>, <article>, <aside>, and <footer>. Generic tags such as <div> and <span> are still useful when no semantic tag fits the purpose.

Using semantic tags correctly creates cleaner code than building every part of the page with only <div> elements.

Website Header

This area usually contains the logo, title, or top navigation.

Navigation helps users move between important sections and pages.

Main Section

A section groups content around one topic.

Article Example

An article is useful for self-contained content like a blog post or news story.

An aside holds related but secondary information such as tips or side notes.

The footer usually contains copyright, links, or contact information.
